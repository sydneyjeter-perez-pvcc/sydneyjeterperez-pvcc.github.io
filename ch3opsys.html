Operating Systems Terminology
Terms Definitions
Operating systems The software that controls the overall operation of the computer including the management, the user interface, and the program execution environment.
Job queue A storage organization in which jobs are ordered in first-in, first-out fashion.
FIFO First-in, first-out; objects are removed from the queue in the order in which they arrived.
Interactive processing A program being executed to carry on a dialogue with the user through remote terminals
Time-sharing An operating system that provided service to multiple users at the same time.
Multiprogramming Time is divided into intervals and then the execution of each job is restricted to only one interval at a time.
Application software Consists of programs for performing tasks particular to the machine's utilization.
System software Performs tasks that are common to the computer systems in general.
Utility software Consists of programs for performing activities fundamental to computer installation but not included in the operation system.
Graphical user interface (GUI) A modern system in which objects can be manipulated, such as files and programs, through pictoral representation display icons.
Shell A user interface that communicatess with users through textual massages using a keyboard and monitor screen.
Kernel The internal part of an operating system.
Device drives/td> A component of a kernel which are a software units that communicate with the controllers (or directly with peripheral devices at times) to carry out operations on the peripheral devices attached to the machine.
File manager A component of a kernal whose job is to coordinate the use of the machine's mass storage facilities.
Memory manager A kernal component charged with the task of coordinating the machine's use of main memory.
Virtual memory Large "fictional" memory space created by paging.
Boot strapping A procedure that transfers the operating system from mass storage into main memory; also known as booting or turning on the system.
ROM Known as read-only memory, this is where the CPU expects to find its initial program that has been constructed fromspecial nonvolatile cells. Its contents can be read but not altered.
Deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
Spooling A technique of holding data for output at a later but more convenient time.
Super user (or administration) A person with high privileges to gain access to the operating system by identifying themselves as the administrator during the login process through an established account.
Multi-factor authentication When a user provides a password as well as additional evidence to gain access to an account.
