Chapter 6: Programming Languages
Terms Definitions
Machine language A computer programming lanuage consisting of binary or hexadecimal instructions which a computer can respond to directly.
Debugging Locating and correcting errors in programs.
Assembler Converts mnemonic expressions into machine language instructions.
Assembly language A mnemonic system for representing programs, also known as second-generation languages.
Translator Translates programs expressed in high-level primitives into machine-language programs.
Complier Translation programs that compiled several machine instructions into short sequences to simulate the activity requested by a single high-level primitive.
Interpreter Implemented third-generation languages and executed the instructions as they were translated instead of just recording it.
Cross-Platform Software Software that is independent of an operating system's design as well as the machine's hardware design.
Natural language Languages such as English, German, and Latin.
Formal language Programming languages.
Machine independence Software that is not dependent on the properties of a particular machine.
High-level progrgamming language A programming language with strong abstraction from the details of the computer.
Object-oriented programming (OOP) A programming paradigm based on the concept of objects, which can contain data and code.
Scripting language A subset of imperative programming languages that are used to perform administrative tasks rather than to develop complex programs.
Data types
  • Integer: Numeric data consisting of whole numbers.
  • Float: Also known as real, refers to numeric data that might contain values other than whole numbers.
  • String: A sequence of characters and the most commonly used data type to store text.
Primitive data type A set of basic data types from which all other data types are constructed.
Data structure The conceptual shape or arrangement of data.
Python collection data structures
  • List: Used to store multiple items in a single variable. These are created by using square brackets.
  • Tuple: A collection which is ordered and unchangeable.
  • Set: A collection which is unordered, unchangeable (you can remove and add new items but not change the items themselves), and unindexed
  • Dictionary: Used to store data values in key:value pairs. They are ordered (Python version 3.7), changeable, and do not allow duplicates.
Function A set of instruction for performing a task that can be used as an abstract tool by other programs.
Local variable A variable declared within a function. It can be reference only within that function.
Global variable Variables whose scopes are not restricted to a particular part of a program.
Source program A program written in a language other than machine code, typically a high-level language.
Object program A computer program translated from the equivalent source program into machine language by the complier or assembler.
