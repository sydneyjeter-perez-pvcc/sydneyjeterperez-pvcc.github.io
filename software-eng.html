Chapter 7: Software Engineering
Terms Definitions
Software engineering
Integrated Development Environment (IDE) Combine tools for developing software (editors, compilers, debugging tools, and so on) into a single, integrated package.
Association for Computing Machinery (ACM)
  • Purpose of ACM: To strengthen profression's collective voice through strong leadership, promotion of the highest standards, and recognition of technical excellence.
  • Benefits of ACM membership: Advancing your career and enriching your knowledge with life-long learning resources.
  • Student ACM membership: Baseline costs $19
Institute of Electrical and Electronics Engineers (IEEE)
  • Purpose of IEEE: To foster technological innovation and excellence for the benefit of humanity.
  • Benefits of IEEE membership: Networking with other technology professors, creating a profile of your highlighting accomplishments, and participate in discussions on various technical interests.
  • Student IEEE membership: Costs $32 and holds a requirement of at least 50% of a normal full-time academic program as a registered undergraduate or graduate student in a regular course of study in IEEE designated fields.
Software Life Cycle: Development phase
  • Requirements analysis: specify what services the proposed system will provide, to identify conditions on those services, and to define how the outside world will interact with the system.
  • Design: creating a plan for the construction of the proposed system.
  • Implementation: Actual writing of the programs, creation of the data files, and development of databases.
  • Testing: Proces of debuggiing programs and confirming that the final software product is compatible with the software requirements specification.
Modular implementation The process of dividing a system into independent modules that can be used to create different configurations.
Unified Modeling Language (UML) A standardized visual modeling language used by software developmers to design and document the structure and behavior of a software system.
Use case A detailed description of how a system is used to achieve a goal, and the interaction between the system and user.
Use case diagram A visual representation of how users (also called actors) interact with a system, hightlighting the different functionalities and scenarios they can utilize within the system.
Data response
  • One-to-one: Data structure where each unique identifier in one data set is associated with exactly one corresponding unique identifier in another data set.
  • One-to-many: Data structure where one single record or entity in a database can be associated with multiple related records or entities in abother table.
  • Many-to-many: A single data record in one system can be associated with multiple records in another system, and vice versa. Both entities can have numerous connections with one another.
Design pattern A reusable solution to a commonly occurring problem in software design.
Software quality assurance (SQA) A set of practices and methodologies that monitor and improce the software development process to ensure the quality and reliabiliy of the software.
Software testing
  • Glass box testing: Software testing method that examines the internal structure and logic of a program.
  • Black box testing: Assesses a program's functionality without knowing its internal code or workings.
  • Beta testing: A pre-release phase where a nearly completed software product is distributed to a limited group of external users, allowing them to test the software in real-world conditions and provide feedback to developers before the official launch.
  • Alpha testing: The intial phase of software testing performed by the development team internally, where the primary goal is to identify and fix bugs and issues within the software before it is released to external users.
Documentation
  • User documentation: A collection of written materials that help users understand and use a software application.
  • System documentation: A type of technical documentation that provides information about a software system's design, architecture, and components.
  • Technical documentation: A collection of documents that outline how to design, build, use, and maintain software.
Intellectual property law
  • Software license: A legally binding agreement between a software creator and the user that defines how the software can be used.
  • Software disclaimer: A legal statement that limits the liability of the entity that created the software. Helps protect them from legal claims and other issues.
  • Best defenses against software liability claims: To apply sound software engineering principles during the software's development, to use a level of care compatible with the software's application, and to produce and maintain records that validate these endeavors.
